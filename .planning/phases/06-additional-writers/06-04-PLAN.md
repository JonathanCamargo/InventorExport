Create MuJoCo writer with MJCF structure (different from URDF/SDF).

Module docstring explaining:
- MuJoCo 3.x MJCF format
- Position: meters (no conversion needed from IR)
- Rotation: quaternion (w,x,y,z) - scalar first (MuJoCo convention)
- Assets: meshes and materials defined separately, referenced by name
- worldbody: base frame is implicit (no explicit base_link needed)

```python
        # 2. Convert meshes (STEP -> STL)
        # 3. Build XML tree
        # 4. Write to file with pretty printing
```

XML structure (per RESEARCH.md):
```xml
<mujoco model="{model.name}">
  <!-- Compiler settings -->
  <compiler meshdir="meshes"/>

  <!-- Asset definitions -->
  <asset>
    <!-- Mesh assets (one per unique part) -->
    <mesh name="{body.name}_mesh" file="{body.name}.stl"/>

    <!-- Material assets -->
    <material name="{material.name}" rgba="0.7 0.7 0.7 1"/>
  </asset>

  <!-- World body containing all rigid bodies -->
  <worldbody>
    <!-- Bodies attached directly to world (no base_link needed) -->
    <body name="{body.name}" pos="{x} {y} {z}" quat="{w} {x} {y} {z}">
      <inertial pos="0 0 0" mass="{mass}" diaginertia="{ixx} {iyy} {izz}"/>
      <geom type="mesh" mesh="{body.name}_mesh" material="{material.name}"/>
    </body>
  </worldbody>
</mujoco>
```

Key differences from URDF/SDF:
1. No explicit base_link - worldbody IS the world frame
2. Assets section defines meshes and materials separately
3. Meshes referenced by name, not path
4. Quaternion orientation (w,x,y,z) instead of RPY
5. `<compiler meshdir="meshes"/>` sets mesh search path
6. Inertia can use diaginertia (diagonal only) or fullinertia (all 6)
7. Bodies directly under worldbody (flat, not tree structure needed for rigid assembly)

Use rotation_to_quaternion(matrix, scalar_first=True) for orientation.

For inertia tensor:
- If off-diagonal terms are zero (within tolerance): use diaginertia="{ixx} {iyy} {izz}"
- Otherwise use fullinertia="{ixx} {iyy} {izz} {ixy} {ixz} {iyz}"

Handle edge cases:
- Bodies without geometry_file: skip geom element
- Bodies without inertia: skip inertial element (MuJoCo will compute from geom)
- Bodies without material_name: omit material attribute on geom
