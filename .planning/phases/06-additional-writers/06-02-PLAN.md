XML structure (per RESEARCH.md):
```xml
<robot name="{model.name}">
  <!-- Materials section -->
  <material name="steel">
    <color rgba="0.7 0.7 0.7 1.0"/>
  </material>

  <!-- Virtual base at world origin -->
  <link name="base_link"/>

  <!-- Physical links -->
  <link name="{body.name}">
    <inertial>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <mass value="{inertia.mass}"/>
      <inertia ixx="..." ixy="..." ixz="..." iyy="..." iyz="..." izz="..."/>
    </inertial>
    <visual>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <mesh filename="meshes/{body.name}.stl"/>
      </geometry>
      <material name="{body.material_name}"/>
    </visual>
    <collision>
      <origin xyz="0 0 0" rpy="0 0 0"/>
      <geometry>
        <mesh filename="meshes/{body.name}.stl"/>
      </geometry>
    </collision>
  </link>

  <!-- Fixed joints connecting bodies to base -->
  <joint name="{body.name}_joint" type="fixed">
    <parent link="base_link"/>
    <child link="{body.name}"/>
    <origin xyz="{pos[0]} {pos[1]} {pos[2]}" rpy="{rpy[0]} {rpy[1]} {rpy[2]}"/>
  </joint>
</robot>
```

Key implementation details:
1. Use lxml.etree.Element/SubElement for XML construction
2. Use rotation_to_euler(matrix, EulerConvention.URDF_RPY, degrees=False) for RPY
3. Position is in meters (IR already uses meters)
4. Use MeshConverter for STEP->STL conversion
5. Inertia tensor values from body.inertia (already in kg*m^2)
6. Material color: derive from density or use sensible defaults (gray for steel)
7. Include XML declaration and pretty print output

Handle edge cases:
- Bodies without geometry_file: skip mesh geometry element
- Bodies without inertia: skip inertial element
- Bodies without material_name: skip material reference

Add verbose comments in output XML (per CONTEXT.md decision).
