XML structure (per RESEARCH.md):
```xml
<sdf version="1.8">
  <model name="{model.name}">
    <!-- Virtual base at world origin -->
    <link name="base_link"/>

    <!-- Physical links -->
    <link name="{body.name}">
      <pose relative_to="base_link">{x} {y} {z} {roll} {pitch} {yaw}</pose>
      <inertial>
        <mass>{inertia.mass}</mass>
        <inertia>
          <ixx>{ixx}</ixx><ixy>{ixy}</ixy><ixz>{ixz}</ixz>
          <iyy>{iyy}</iyy><iyz>{iyz}</iyz><izz>{izz}</izz>
        </inertia>
      </inertial>
      <visual name="visual">
        <geometry>
          <mesh><uri>meshes/{body.name}.stl</uri></mesh>
        </geometry>
      </visual>
      <collision name="collision">
        <geometry>
          <mesh><uri>meshes/{body.name}.stl</uri></mesh>
        </geometry>
      </collision>
    </link>

    <!-- Fixed joints connecting bodies to base -->
    <joint name="{body.name}_joint" type="fixed">
      <parent>base_link</parent>
      <child>{body.name}</child>
    </joint>
  </model>
</sdf>
```

Key differences from URDF:
1. Root element is `<sdf version="1.8">` containing `<model>`
2. Pose uses single `<pose>` element with 6 space-separated values: x y z roll pitch yaw
3. Inertia elements are nested differently (separate elements, not attributes)
4. Mesh uses `<uri>` child element instead of filename attribute
5. Visual/collision have name attribute

Use same rotation conversion as URDF: rotation_to_euler(matrix, EulerConvention.URDF_RPY, degrees=False)

Handle edge cases same as URDF writer:
- Bodies without geometry_file: skip mesh geometry
- Bodies without inertia: skip inertial element
- Bodies without material_name: skip material
