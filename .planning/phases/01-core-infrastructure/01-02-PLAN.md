<objective>
Create rotation conversion utilities using scipy.spatial.transform.Rotation.

Purpose: Inventor stores orientations as 3x3 rotation matrices. Export formats require different representations: ADAMS uses ZXZ Euler angles, URDF uses RPY (roll-pitch-yaw), MuJoCo uses quaternions. Using scipy avoids gimbal lock bugs present in the existing VBA code.

Output: Rotation module that correctly converts matrices to all target format conventions, verified against known test matrices.
</objective>

   - Use `warnings.catch_warnings(record=True)` to capture gimbal lock warnings
   - Call `r.as_euler(convention.value, degrees=degrees)`
   - Log warning if gimbal lock detected
   - Return tuple of three angles

4. `rotation_to_quaternion(rotation_matrix: np.ndarray, scalar_first: bool = True, canonical: bool = True) -> Tuple[float, float, float, float]`:
   - Create Rotation from matrix
   - Call `r.as_quat(canonical=canonical, scalar_first=scalar_first)`
   - Return tuple of four values (w, x, y, z if scalar_first=True)

5. Logger: `logger = logging.getLogger(__name__)`
