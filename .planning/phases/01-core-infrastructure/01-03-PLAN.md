<objective>
Create COM connection utilities with proper lifecycle management for Inventor automation.

Purpose: Python's garbage collector doesn't deterministically release COM objects, leading to memory leaks and stale references. Context managers ensure COM objects are released even on exceptions, preventing "object disconnected" errors and memory growth.

Output: COM module with inventor_app() and active_assembly() context managers that connect to Inventor cleanly and release resources on exit.
</objective>

  <action>
Create `src/inventor_exporter/core/com.py` implementing:

1. Imports:
   - `import pythoncom`
   - `import win32com.client`
   - `from contextlib import contextmanager`
   - `from typing import Generator, Any`
   - `import logging`

2. Custom exceptions:
   - `class InventorNotRunningError(Exception)`: Raised when Inventor is not running
   - `class NotAssemblyError(Exception)`: Raised when active document is not an assembly

3. `@contextmanager inventor_app() -> Generator[Any, None, None]`:
   - Get logger: `logger = logging.getLogger(__name__)`
   - Log debug: "Connecting to Inventor..."
   - Try `win32com.client.GetActiveObject("Inventor.Application")`
   - On success, log debug with version: `app.SoftwareVersion.DisplayVersion`
   - Yield the app object
   - In finally block: if app is not None, log debug "Releasing Inventor connection" and delete reference
   - Catch `pythoncom.com_error` with hresult -2147221021 (MK_E_UNAVAILABLE) and raise `InventorNotRunningError` with helpful message

Follow the exact pattern from 01-RESEARCH.md "Complete COM Context Manager" section.

Note: Do NOT call pythoncom.CoInitialize() in the context manager - it's auto-initialized on main thread. Only worker threads need explicit initialization (not implemented in this phase).
