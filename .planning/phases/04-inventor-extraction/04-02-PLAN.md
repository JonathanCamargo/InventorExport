```python
"""STEP geometry export via Inventor TranslatorAddIn.

Uses the official TranslatorAddIn mechanism with GUID {90AF7F40-0C01-11D5-8E83-0010B541CD80}
for STEP export. Exports each unique part definition once (not per occurrence).
"""
```

**Constants:**
```python
STEP_TRANSLATOR_GUID = "{90AF7F40-0C01-11D5-8E83-0010B541CD80}"

# Application Protocol Types
AP203 = 2   # Configuration Controlled Design
AP214 = 3   # Automotive Design (better compatibility)
AP242 = 5   # Managed model based 3D engineering (newest)
```

**export_step(app, document, output_path: Path, protocol: int = AP214) -> bool:**
- Get translator: `app.ApplicationAddIns.ItemById(STEP_TRANSLATOR_GUID)`
- Cast to TranslatorAddIn: `win32com.client.CastTo(translator, "TranslatorAddIn")`
- Create translation context: `app.TransientObjects.CreateTranslationContext()`
- Set context.Type = 13059 (kFileBrowseIOMechanism)
- Create options: `app.TransientObjects.CreateNameValueMap()`
- If `translator.HasSaveCopyAsOptions(document, context, options)`:
  - Set `options.Value("ApplicationProtocolType") = protocol`
- Create data medium: `app.TransientObjects.CreateDataMedium()`
- Set `data_medium.FileName = str(output_path)`
- Call `translator.SaveCopyAs(document, context, options, data_medium)`
- Return True on success, False on error (with logging)

**export_unique_parts(app, occurrences: list, output_dir: Path) -> dict[str, Path]:**
- Takes list of occurrence data (with definition_path and part_document)
- Deduplicates by definition_path (document full filename)
- For each unique part:
  - Generate output filename from part name (sanitize)
  - Call export_step()
  - Map definition_path -> exported STEP path
- Return mapping dict

Important notes from research:
- CastTo is required - ItemById returns base ApplicationAddIn interface
- Without cast, AttributeError: 'COMObject' has no attribute 'SaveCopyAs'
- Output filename should use sanitized part name (replace colons/spaces)
