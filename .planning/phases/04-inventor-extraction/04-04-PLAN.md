**InventorClient class:**

```python
class InventorClient:
    """High-level client for extracting assembly data from Inventor.

    Example:
        client = InventorClient()
        model = client.extract_assembly(output_dir=Path("./output"))
        errors = model.validate()
    """

    def __init__(self):
        """Initialize client. Does NOT connect to Inventor yet."""
        self._app = None
        self._doc = None
        self.logger = logging.getLogger(__name__)
```

**extract_assembly(output_dir: Path) -> AssemblyModel:**

This is the main entry point. Steps:

1. **Connect to Inventor:**
   - Use `inventor_app()` context manager from core/com.py
   - Use `active_assembly()` to get assembly document
   - Store references for use throughout extraction

2. **Traverse assembly:**
   - Call `traverse_assembly(doc)` from assembly.py
   - Get list of OccurrenceData

3. **Export STEP geometry:**
   - Call `export_unique_parts(app, occurrences, output_dir)` from geometry.py
   - Get mapping: definition_path -> STEP file path

4. **Extract materials (deduplicated):**
   - Track unique materials by name
   - For each unique part_document:
     - Call `extract_material(part_doc)` from material.py
     - Add to materials dict if not duplicate
   - Convert to tuple of Material objects

5. **Build bodies:**
   - For each OccurrenceData:
     - Get STEP path from geometry mapping (by definition_path)
     - Get material from extracted materials
     - Call `extract_mass_properties(part_definition)` from mass.py
     - Create Body with:
       - name (from occurrence)
       - transform (from OccurrenceData)
       - material_name (reference, not object)
       - inertia (from mass properties)
       - geometry_file (STEP path)

6. **Build AssemblyModel:**
   - name from doc.DisplayName
   - bodies tuple
   - materials tuple

7. **Validate and return:**
   - Call model.validate()
   - Log any validation errors
   - Return model

**Error handling:**
- Wrap in try/finally for COM cleanup
- Catch and log individual extraction errors without aborting
- Parts with extraction errors get logged but skipped
- Return partial model if some parts fail

Implementation notes:
- Use context managers for COM lifecycle
- Delete COM references after use
- Log progress (part X of Y)
- Handle case where material or mass props extraction fails for a part

from Autodesk Inventor via COM automation.

Classes:

Functions:
    traverse_assembly: Walk assembly hierarchy, collect leaf occurrences.
    extract_transform: Get Transform from occurrence.
    export_step: Export single document to STEP format.
    export_unique_parts: Export STEP for unique part definitions.
    extract_material: Get Material from part document.
    extract_mass_properties: Get Inertia from part definition.
