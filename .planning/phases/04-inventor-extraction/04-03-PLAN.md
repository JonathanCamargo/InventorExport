```python
"""Material property extraction from Inventor parts.

Reads material name and density from the ActiveMaterial's PhysicalPropertiesAsset.
Handles localization issues by searching for partial property name matches.
"""
```

**Unit conversion constant:**
```python
# Density conversion: kg/cm^3 to kg/m^3
# 1 kg/cm^3 = 1,000,000 kg/m^3
CM3_TO_M3 = 1_000_000
```

**extract_material(part_doc) -> Optional[Material]:**
- Get `material = part_doc.ActiveMaterial`
- If material is None: log warning, return None
- Get `material_name = material.DisplayName`
- Get `phys_props = material.PhysicalPropertiesAsset`
- If phys_props is None: log warning, return Material with name only and default density
- Search for density property:
  - Iterate `for prop in phys_props`
  - Check `if "density" in prop.Name.lower() or "dichte" in prop.Name.lower()` (German support)
  - Get `density_kg_cm3 = prop.Value`
- If density found:
  - Convert: `density_kg_m3 = density_kg_cm3 * CM3_TO_M3`
  - Return `Material(name=material_name, density=density_kg_m3)`
- If not found: log warning, return Material with name and fallback density (7800 for steel)

Important notes from research:
- Property names vary by locale ("Density" in English, "Dichte" in German)
- Use partial match, not exact match
- Always return something valid (Material requires positive density)
- Log warnings for missing data, don't fail
  </action>

Expected: `CM3_TO_M3=1000000`
  </verify>
  <done>
- extract_material() reads from ActiveMaterial.PhysicalPropertiesAsset
- Density converted from kg/cm^3 to kg/m^3

Create `extraction/mass.py` for mass and inertia extraction:

```python
"""Mass property extraction from Inventor parts.

Uses ComponentDefinition.MassProperties to get mass, center of mass, and
the full 6-component inertia tensor via XYZMomentsOfInertia.
"""
```

**Unit conversion constant:**
```python
# Inertia conversion: kg*cm^2 to kg*m^2
# cm^2 = 0.0001 m^2
CM2_TO_M2 = 0.0001
```

**extract_mass_properties(part_definition) -> Inertia:**
- Get `mp = part_definition.MassProperties`
- Get mass: `mass = mp.Mass` (already in kg)
- Get center of mass: `com = mp.CenterOfMass`
  - Convert to meters: use InventorUnits.length_to_meters() for com.X, com.Y, com.Z
  - Create numpy array
- Get inertia tensor:
  - Call `result = mp.XYZMomentsOfInertia()` - returns tuple in Python (not ByRef like VBA)
  - Unpack: `Ixx, Iyy, Izz, Ixy, Iyz, Ixz = result`
  - Convert each from kg*cm^2 to kg*m^2 (multiply by CM2_TO_M2)
  - Construct 3x3 symmetric tensor:
    ```python
    inertia_tensor = np.array([
        [Ixx * CM2_TO_M2, Ixy * CM2_TO_M2, Ixz * CM2_TO_M2],
        [Ixy * CM2_TO_M2, Iyy * CM2_TO_M2, Iyz * CM2_TO_M2],
        [Ixz * CM2_TO_M2, Iyz * CM2_TO_M2, Izz * CM2_TO_M2],
    ])
    ```
- Return `Inertia(mass=mass, center_of_mass=center_of_mass, inertia_tensor=inertia_tensor)`

Important notes from research:
- XYZMomentsOfInertia in pywin32 returns a tuple (not ByRef parameters like VBA)
- Tensor is symmetric: Ixy appears at [0,1] and [1,0]
- Sign convention: products of inertia may be negative
- Inertia is about coordinate system origin, NOT center of mass (may need at_point() later)
  </action>
  <verify>
Run: `python -c "from inventor_exporter.extraction.mass import extract_mass_properties, CM2_TO_M2; print(f'CM2_TO_M2={CM2_TO_M2}')"`
Expected: `CM2_TO_M2=0.0001`
  </verify>
  <done>
- extract_mass_properties() uses MassProperties.XYZMomentsOfInertia()
- Returns Inertia dataclass with full 6-component tensor
- Units converted from cm to m
- Handles pywin32 tuple return (not VBA ByRef)
  </done>
