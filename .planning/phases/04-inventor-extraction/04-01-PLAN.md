2. `extraction/assembly.py` - Assembly traversal and transform extraction:

```python
"""Assembly traversal and transform extraction from Inventor.

Uses AllLeafOccurrences for flat traversal - this property returns all leaf
parts at any nesting level with transformations already in assembly (world)
coordinates. No manual transform accumulation needed.
"""
```

Key functions:

**OccurrenceData dataclass:**
- name: str (occurrence name)
- transformation: Transform (position + rotation in world frame)
- definition_path: str (document path for deduplication)
- part_document: Any (COM reference for material/mass extraction)

**extract_transform(occurrence) -> Transform:**
- Get `occurrence.Transformation` matrix (4x4 COM object)
- Extract translation from Cell(4,1), Cell(4,2), Cell(4,3) - units are cm
- Convert to meters using `InventorUnits.length_to_meters()`
- Extract rotation using `extract_rotation_matrix()` from core/rotation.py
- Return Transform dataclass

**traverse_assembly(asm_doc) -> list[OccurrenceData]:**
- Get `asm_def = asm_doc.ComponentDefinition`
- Get `leaf_occs = asm_def.Occurrences.AllLeafOccurrences`
- Log count: `logger.info(f"Found {leaf_occs.Count} leaf occurrences")`
- For each occurrence:
  - Extract transform using extract_transform()
  - Get definition_path from `occ.Definition.Document.FullFileName`
  - Get part_document from `occ.Definition.Document`
  - Create OccurrenceData
  - Delete occurrence reference after processing (COM cleanup)
- Return list of OccurrenceData

Important implementation notes from research:
- Inventor Matrix.Cell() uses 1-based indexing
- Translation is in row 4: Cell(4,1), Cell(4,2), Cell(4,3)
- Rotation is in upper-left 3x3: Cell(1..3, 1..3)
- AllLeafOccurrences Transformation already includes all parent transforms
- Units are always cm regardless of document settings
